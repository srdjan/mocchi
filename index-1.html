Semantic HTML, Minimal Classes

sem-mochi leverages modern CSS capabilities and semantic HTML to minimize class usage while maintaining sophisticated styling capabilities.

Semantic Benefit: Cleaner markup, better accessibility, improved maintainability, and enhanced SEO.

Semantic-First Design

Styles are applied directly to HTML elements based on their semantic meaning, minimizing the need for utility classes.

Leverages modern CSS selectors and cascade layers to avoid specificity conflicts.

Attribute-Based Variants

Component variations use HTML attributes instead of modifier classes:

<button primary>Primary</button>
<button outline>Outline</button>

ARIA-Driven States

Interactive states are managed through ARIA attributes rather than state classes:

<button aria-expanded="true">
  Menu
</button>

Fundamental Principles

  1. Write semantic HTML first - Focus on the meaning of your content
  2. Use attributes for variants - Apply component variations with HTML attributes
  3. Leverage ARIA for states - Use standardized ARIA attributes for interactive states
  4. Minimize classes - Only use classes for true composition patterns
